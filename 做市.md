# 稳定币交易对做市服务

我发行的美元稳定币（记作 USDx）/USDT，在 Uniswap v3 上完整做市架构

### 目标

- **把 USDx/USDT 价格稳定在 1.000 附近**（例如 ±5~20 bps）
- **提供可观深度**（1.000 附近大量流动性）
- **赚取手续费**（在波动/交易量存在时）
- **限制尾部风险**（脱锚、单边吃光、恶意砸盘、MEV）

### 关键约束（稳定币池子的特性）

- **波动很小** → v3 的集中流动性优势非常明显（可以把资金集中在 1 附近）
- **脱锚时风险极大** → 必须有“熔断/撤池/限额/对冲”机制
- **AMM被动 LP 不等于做市** → 需要再平衡、库存管理、资金补给、事件响应

## 系统架构（模块化）

### 核心模块清单

**Uniswap v3 Pool（USDx/USDT）**

- 创建交易对，设定 fee tier（0.05%）
- 初始化 price（必须 1:1 附近）

**LP Position Manager（头寸管理器）**

- 负责：开仓、加/减流动性、收取 fees、关闭头寸
- 可选实现方式：
  - 直接调用 Uniswap v3 NonfungiblePositionManager

**Rebalancer Bot（再平衡机器人）**

- 核心“做市逻辑”
- 负责：根据价格偏离、库存、费用收益、gas、波动来调整区间、重新部署流动性

**Treasury / Inventory（资金与库存系统）**

- 两个资金池概念：
  - **LP 资金池**：用于做 v3 LP
  - **Stabilization 资金池**：用于脱锚时的主动稳定（swap 回拉/回购/补保证金）
  
  

**Price Oracle & Reference（价格参考）**

- **链上**：Uniswap v3 TWAP（反操纵）



**Execution Layer（交易执行层）**

- 交易路径：Rebalancer → Router（如 Universal Router）→ RPC → 链上
- 建议集成：
  - **私有交易通道**（Flashbots / builder relay）降低被夹/被抢跑概率
  - 失败回滚与重试策略（nonce 管理、gas 策略）

1. **Risk Engine（风控引擎）**

- 决策：是否撤池、是否扩区间、是否暂停增仓、单次最大投入、单日最大损失
- 触发：脱锚、交易量异常、MEV 异常、合约风险、oracle 偏离

1. **Observability（监控告警）**

- 指标：价格偏离、区间覆盖率、未实现损益、fees APR、库存占比、gas、失败率、池子深度、TWAP 偏差
- 告警：脱锚、头寸离开区间、库存单边、失败率飙升、被攻击迹象



### 头寸结构：三层流动性（推荐）

把资金拆成三档，形成“深度 + 缓冲 + 尾部防御”。

1. **Core（核心深度层）**

- 区间：例如 **[0.9995, 1.0005]**（±5 bps）或 ±10 bps
- 占比：40%~70%
- 目的：让交易者在 1 附近几乎“无滑点”

1. **Mid（缓冲层）**

- 区间：例如 **[0.995, 1.005]**（±50 bps）或 ±100 bps
- 占比：20%~40%
- 目的：减少频繁再平衡；应对轻微脱锚或波动扩大

1. **Tail（尾部防御层）**

- 区间：例如 **[0.98, 1.02]** 或更宽
- 占比：5%~20%
- 目的：极端情况下还能持续提供一些报价，避免瞬间枯竭，同时为你争取时间触发稳定机制/撤池

> 稳定币对的核心思想：**“窄区间赚手续费 + 宽区间保连续性 + 风控负责极端情况”**





##  主动稳定机制（脱锚时：LP 不够用）

仅靠 v3 LP，有可能在脱锚时被“吃光”而无法回拉。你需要一个 **Stabilization Engine**：

1. **链上主动 swap 回拉**

- 当 USDx < 1：用 USDT 买入 USDx（提高价格）
- 当 USDx > 1：卖出 USDx 换 USDT（压回价格）
- 这部分资金来自 Stabilization 资金池



## 风控与安全（决定你能不能长期跑）

### 6.1 熔断策略（建议硬编码在管理合约/风控层）

- **TWAP 脱锚熔断**：例如 5~15 分钟 TWAP 偏离 > 30~100 bps
   → **暂停增仓、收缩 Core、甚至撤出全部流动性**
- **单日最大损失**：达到阈值自动停止
- **单次最大投入**：每次 rebalance 最大动用资金比例
- **异常交易量/滑点**：短时间大成交 + 价格大跳 → 进入保护模式

### 6.2 MEV 防护（稳定币池也会被夹）

- 再平衡/主动 swap 尽量走 **私有交易通道**
- 所有交易设置：
  - 最大滑点
  - deadline
  - 分批执行（大额拆单）
- 避免在“显而易见的可被抢跑窗口”公开广播大额调仓



## 监控指标与告警（上线后最关键）

### 必备仪表盘指标

- 价格偏离：pool spot、TWAP、refPrice 三者
- 深度：1.000 ± 10/50/100 bps 的可交易量
- 头寸状态：是否在区间内、各层占比、未实现损益
- 库存：USDx/USDT 余额、目标比例偏差
- 收益：fees、gas 成本、净收益
- 执行：交易成功率、失败原因、nonce 堵塞
